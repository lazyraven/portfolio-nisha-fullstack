Interview Example

🎯 Two Sum — Optimal Solution

twoSum.js — HashMap approach
Time: O(n) Space: O(n) 
// Find two numbers that add up to target
function twoSum(nums, target) {
  const map = new Map();   // store: num → index

  for (let i = 0; i < nums.length; i++) {
    let complement = target - nums[i]; // what we need

    if (map.has(complement)) {         // found it!
      return [map.get(complement), i];
    }

    map.set(nums[i], i);             // store current
  }
}

// twoSum([2,7,11,15], 9) → [0, 1]

🌳 Trees

DOM · File Systems · Decision Trees
Traversal Patterns
DFS — Depth First
  • ▸Inorder: Left → Root → Right (BST sorted)
  • ▸Preorder: Root → Left → Right (copy tree)
  • ▸Postorder: Left → Right → Root (delete tree)
  • ▸Recursive and iterative approaches
  • ▸Uses call stack — O(h) space where h = height
Level Order
BFS — Breadth First
  • ▸Process nodes level by level
  • ▸Uses a Queue — FIFO
  • ▸Level order traversal output
  • ▸Right-side view of tree
  • ▸Find minimum depth
BST & Practice
Top Tree Problems
  • ▸Maximum Depth of Binary Tree
  • ▸Lowest Common Ancestor (LCA)
  • ▸Validate BST — inorder should be sorted
  • ▸Symmetric Tree — mirror check
  • ▸Path Sum — root-to-leaf

Phase 6 — Game Changer 🔥

⚡ Dynamic Programming

Optimization · AI/Game Logic · Resource Allocation
Top-Down
Memoization
  • ▸Recursion + cache results
  • ▸Use HashMap / array as memo
  • ▸Solve sub-problem once, reuse
  • ▸More intuitive — mirrors recursion
  • ▸Top-down = recursive + cache
Bottom-Up
Tabulation
  • ▸Fill DP table iteratively
  • ▸No recursion — avoids stack overflow
  • ▸Usually more space-efficient
  • ▸1D DP: Climbing Stairs, Fibonacci
  • ▸2D DP: Longest Common Subsequence
Classic DP Problems
Must-Solve DP
  • ▸Climbing Stairs — Fibonacci DP
  • ▸Longest Increasing Subsequence (LIS)
  • ▸0/1 Knapsack — include/exclude choice
  • ▸Coin Change — minimum coins
  • ▸Edit Distance — 2D DP
⚠️
Don't jump to DP early.Master arrays, trees, and graphs first — DP builds on those foundations.

⚠️ Watch Out

Common Mistakes to Avoid

Jumping to DP Too Early
DP requires strong recursion + array foundations. Master simpler patterns first — arrays, trees, then DP.
Not Revising Problems
Re-solve problems after 1 week. Spaced repetition cements patterns better than solving new problems daily.
Memorizing Instead of Understanding
Interviewers change inputs. Understanding why an approach works beats memorizing the code every time.
