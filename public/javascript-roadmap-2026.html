⚙️ Functions & Advanced Concepts

Interview Core ⭐
Closures & Scope
  • ▸Closures — function + its lexical scope
  • ▸Lexical Scope — where it's defined
  • ▸Higher-Order Functions
  • ▸Callbacks — passing functions as args
  • ▸Practice: Counter using closure
Why asked?
Closures appear in debounce, memoization, module patterns, and private state — core to JS mastery.
Critical for Interviews
this keyword
  • ▸this in global, method, arrow context
  • ▸bind() — permanently set context
  • ▸call() — invoke with context + args
  • ▸apply() — invoke with context + array
  • ▸Practice: Custom bind() implementation
Performance
Debounce & Throttle
  • ▸Debounce: delay until input stops
  • ▸Throttle: limit execution rate
  • ▸Real use: search bars, scroll events
  • ▸Combines closures + this
  • ▸Practice: Build both from scratch
// Classic debounce — tests closures + this + real-world thinking function debounce(fn, delay) { let timer; return function (...args) { clearTimeout(timer); timer = setTimeout(() => { fn.apply(this, args); }, delay); }; }

async / await
  • ▸async functions always return Promise
  • ▸await pauses execution until resolved
  • ▸try/catch for error handling
  • ▸Fetch API + Axios patterns
  • ▸Parallel: Promise.all with async/await
Deep Internals ⭐
Event Loop
  • ▸Call Stack → Web APIs → Queue
  • ▸Microtask queue (Promises)
  • ▸Macrotask queue (setTimeout)
  • ▸Microtasks run before macrotasks
  • ▸Single-threaded non-blocking model
Q: What logs first — Promise or setTimeout?
A: Promise (microtask) always before setTimeout (macrotask), even with 0ms delay.
